# Repository Positioning
## Purpose
This document clarifies what this repository is, and what it is not.
Its goal is to prevent misinterpretation of scope, maturity, and intent.
## What this repository is
This repository is a **curated public interface** for a structured research program:
- Canonical Reality Theory (CRT)
- MST–CRT–IET architecture stack
- Determinization, canonicalization, and escape-resistance frameworks
It provides:
- Architecture overview
- Status tracking
- Methodology
- Governance rules
- Publication boundaries
- Citation anchor
It is designed to:
- Establish public authorship
- Maintain claim discipline
- Support future academic publication
- Provide a stable reference point
## What this repository is NOT
This repository is NOT:
- A complete theory archive
- A peer-reviewed publication
- A finished formal system
- An implementation library
- A production system
- An open contribution project
- A raw research notebook
## Maturity level
The research program represented here is:
- Actively evolving
- Partially formalized
- Under ongoing audit
- Not fully published
Claims must be interpreted under STATUS.md and CLAIM_LABELS.md.
## Relationship to private work
There exist private repositories and working materials that are not included here.
These may contain:
- Raw drafts
- Unreleased theorems
- Internal audits
- Experimental structures
Public release is selective and controlled.
## Relationship to future papers
This repository is not a substitute for formal publication.
Future papers (e.g. MST, CRT, IET components) will:
- Provide formal definitions
- Provide proofs
- Provide formal evaluation
This repository acts as:
> a public anchor, not a publication endpoint
## Interpretation rules
Readers must not:
- Treat documentation as theorem statements
- Treat structure descriptions as proofs
- Treat terminology as fully formalized definitions
- Treat repository content as peer-reviewed
## Design philosophy
The repository follows strict separation:
```text
Structure ≠ Proof
Documentation ≠ Formalization
Public layer ≠ Full research
```

This separation is intentional.

Final note

If you are looking for:

* formal proofs → not in this repository
* final theory statements → not guaranteed here
* production-ready systems → not provided

If you are looking for:

* architecture
* structure
* research direction
* authorship anchor

→ this repository is the correct entry point
