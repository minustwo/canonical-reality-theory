# IET Naming
## Purpose
This document resolves a naming conflict in the public CRT research repository.
## Canonical meaning
In this repository, **IET** means:
```text
Inferential Equilibrium Theory

IET is the dynamic adoption and stochastic-selection layer of the MST–CRT–IET stack.

It studies:

* adoption of canonical worlds
* stochastic stability
* process-specific selection
* noisy or perturbed dynamics
* conditions under which canonical worlds may become selected

Boundary

IET is not:

* the institutional application layer
* the governance application layer
* the market application layer
* the AI-system application layer
* a proof of MST
* a proof of CRT
* a universal dynamic canonicality theorem

Legacy naming note

Earlier scaffold material may have used IET to mean one of:

Institutional Escape Theory
Institutional / Interpretive Escape Theory
Institutional / Invariant / Interpretive layer

Those uses are now treated as legacy scaffold language.

They should be read as application-layer or institutional-interpretation material unless explicitly formalized as Inferential Equilibrium Theory.

Current convention

Use:

IET = Inferential Equilibrium Theory

Use:

applications/

for institutional, governance, market, DeFi, AI-system, or organizational interpretations.

Citation rule

Do not cite legacy institutional-IET scaffold files as defining canonical IET.

Canonical IET status should be read through:

* iet/README.md
* docs/ARCHITECTURE.md
* docs/TERMINOLOGY.md
* docs/CLAIMS_LEDGER.md
* papers/foundations/IET_STATUS.md

Status

Status: terminology clarification.

Peer review: not externally peer reviewed.

Claim-status impact: no theorem-status change.
