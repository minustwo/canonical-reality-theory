# Citation Policy
## Purpose
This document defines how this repository and its associated ideas should be cited.
## Core principle
This repository represents an original research program.
Use of ideas, structure, terminology, or derived frameworks must acknowledge origin.
## What requires citation
Citation is required if you use:
- Terminology introduced in this repository (e.g. canonical reality, break path, escape cost)
- Structural frameworks derived from MST–CRT–IET
- Conceptual models based on determinization + canonicalization + escape-resistance
- Any adapted or extended version of the framework
## What does NOT replace citation
The following do not remove the need to cite:
- Rewriting terminology
- Translating the concepts
- Partial reuse of ideas
- Reimplementation in code
- Embedding in another framework
## Citation format
Until formal papers are released, cite as:
Author: Hai Hai Fu  
Project: Canonical Reality Theory (CRT)  
Repository: https://github.com/minustwo/canonical-reality-theory
## Relationship to future publications
Future formal papers (e.g. MST, CRT, IET components) may supersede this repository for citation.
However:
- This repository remains the earliest public anchor
- Citation of origin should still be preserved where appropriate
## Misuse
The following are considered misuse:
- Presenting derived work as original without attribution
- Rebranding core concepts without citation
- Claiming independent discovery of identical framework structures
- Using repository content in academic or commercial work without acknowledgment
## Enforcement
This repository does not enforce citation legally.
However:
- Public record establishes authorship
- Misuse may be challenged in academic or professional contexts
