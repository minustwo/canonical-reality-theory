Open Problems

Purpose

This document records open problems and unresolved dependencies in the MST–CRT–IET research stack.

It does not claim these problems are solved.

OP-1 — MST global completion

Determine conditions under which MST property (4) holds globally, rather than only (4').

Status: OPEN / CONDITIONAL.

Boundary:

* MST v2.16.4 is archival.
* (4') does not automatically imply full (4).

OP-2 — MST language formalization

Develop a typed multi-layer MST language covering:

* Layer-1 admissibility
* Layer-2 justification
* CRT break / escape interfaces
* IET dynamic selection dependencies

Status: OPEN / WORKING.

OP-3 — MST stratified core

Define a stratified MST core separating:

* canonicality at Layer 1
* justification at Layer 2
* robustness / escape at CRT layers
* dynamic adoption at IET layers

Status: OPEN / WORKING.

OP-4 — CRT bridge conditions

Clarify when bridge conditions jointly support full alignment of structural correctness, robustness, and traceability.

Status: CONDITIONAL.

OP-5 — IET process generality

Extend or classify IET beyond reduced sliding-window / process-specific regimes.

Status: OPEN.

Boundary:

* Current IET public status should not be read as full dynamic canonicality.

OP-6 — Full synthesis

Determine whether a full MST–CRT–IET synthesis can be stated under explicit assumptions without overclaim.

Status: OPEN / CONDITIONAL.

OP-7 — Applications boundary

Clarify which institutional, market, governance, DeFi, or AI-system examples are:

* illustrations
* diagnostic artifacts
* formal instantiations
* theorem-supporting evidence

Status: OPEN / DOMAIN-SPECIFIC.
