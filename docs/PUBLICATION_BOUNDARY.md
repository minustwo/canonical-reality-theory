# Publication Boundary
## Purpose
This document defines what may enter the public repository.
## Boundary definition
The repository is a curated public surface.
It is not:
- the full research archive
- the private development workspace
- the raw theorem repository
## Allowed
- Documentation
- Architecture descriptions
- Status tracking
- Policy files
- High-level summaries (non-theorem)
## Restricted
- Raw research notes
- Internal audit logs
- Unreviewed theorem drafts
- Private repo material
- Mixed-language content
## Critical rule
Public release is an explicit decision.
No file crosses the boundary automatically.
## Separation principle
Maintain strict separation:
```text
Private research ≠ Public repository

Violation of this separation breaks the integrity of the project.
